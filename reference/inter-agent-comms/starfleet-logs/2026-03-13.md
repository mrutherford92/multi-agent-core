**Mission**: Audio Playback Bug Fix & Health Check Tooling

### Root Cause Discovery
- Recording files named `.m4a` are actually **WAV (RIFF PCM 16-bit)** data
- iOS `record` plugin saves WAV but uses `.m4a` extension
- iOS AVFoundation reads extension → expects AAC → finds WAV → `-11829 Cannot Open`
- Confirmed: `ffprobe` shows `RIFF (little-endian) data, WAVE audio, Microsoft PCM, 16 bit, mono 44100 Hz`

### Client-Side Fix
- Modified `recording_player_view_model.dart` — magic bytes detection
- Downloads file, reads first 4 bytes: `RIFF` = WAV, `ftyp` = M4A
- Saves cached file with correct extension (`.wav` not `.m4a`)
- Also added: fresh URL fallback for expired Firebase Storage download URLs
- Also changed: `setFilePath()` → `setAudioSource(AudioSource.uri(Uri.file()))` for robustness

### Admin Tooling Built
- **`scripts/audio_health_check.py`** — CLI scanner with `--fix` mode
- **`pages/12_Audio_Health.py`** — Streamlit dashboard page
